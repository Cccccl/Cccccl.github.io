JS的new

  • 不用创建临时对象，因为 new 会帮你做（你使用「this」就可以访问到临时对象）；
  • 不用绑定原型，因为 new 会帮你做（new 为了知道原型在哪，所以指定原型的名字为 prototype）
  • 不用 return 临时对象，因为 new 会帮你做；
  • 不要给原型想名字了，因为 new 指定名字为 prototype。
  • new 操作为了记录「临时对象是由哪个函数创建的」,添加一个constructor 属性

继承

  • 原型链继承
    1. 原型对象
      1. 只要创建一个新函数(Person)，就会有一个prototype属性，这个属性指向构造函数的原型对象(Person.prototype指向构造函数的原型对象(Person Prototype))
      2. 函数的原型对象中会由一个constructor属性，这个属性包含一个指向prototype属性所在函数的指针(Person.prototype.constructor指向Person)
      3. Person构造函数创建一个实例person1，person1内部包含一个指针(内部指针)，指向构造函数的原型对象,指针叫做[[prototype]],(chrome在每个对象上都支持一个属性proto)
      4. 这个连接存在于实例和构造函数的原型对象之间，而不是 实例和构造函数之间
    2. 原型链继承
1.  每个构造函数(Parent)都有一个原型对象(Parent.prototype)，原型对象都包含一个指向构造函数的指针(Parent.prototype有一个指针指向构造函数Parent)，实例中都包含一个指向原型对象的内部指针。(实例temp有一个指针指向原型对象Parent.prototype)
2. 原型对象(Child.prototype)等于另一个构造函数的实际例子(temp由构造函数Parent创建),此时原型对象Child.prototype指向实例temp,实例(temp)又指向构造函数的原型对象(Parent.prototype),那么也就是原型对象Child.prototype指向另一个原型对象Parent.prototype。
3. 另一个原型对象Parent.prototype包含指针指向另一个构造函数Parent，那么这就是一个原型链
