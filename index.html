遍历对象的属性和数组的元素

发表于 2018-05-02 | 分类于 js

数组

  • for loops: for(var i=0;i<arr.length;i++),var是函数作用域而不是块级作用域,大多时候你想要迭代变量在块级作用域中.es2015,y进入了具有块级作用域的let. for(let i =0;i < arr.length; i++)
  • forEach: arr.forEach(function(el,index){})
  • some: some() 方法用于检测数组中的元素是否满足指定条件（函数提供）。
    some() 方法会依次执行数组的每个元素：
    如果有一个元素满足条件，则表达式返回true , 剩余的元素不会再执行检测。
    如果没有满足条件的元素，则返回false。
    注意： some() 不会对空数组进行检测。
    注意： some() 不会改变原始数组。
  • every：every() 方法用于检测数组所有元素是否都符合指定条件（通过函数提供）。
    every() 方法使用指定函数检测数组中的所有元素：
    如果数组中检测到有一个元素不满足，则整个表达式返回 false ，且剩余的元素不会再进行检测。
    如果所有元素都满足条件，则返回 true。
    注意： every() 不会对空数组进行检测。
    注意： every() 不会改变原始数组。

  • Object.getOwnPropertyNames()方法返回对象的所有自身属性的属性名（包括不可枚举的属性）组成的数组，但不会获取原型链上的属性。

浅谈跨域

发表于 2018-05-02 | 分类于 js

AJAX

ajax,就是用js向服务端发起一个请求，并获取服务器返回的内容。
在ajax被发明之前，浏览器发起请求：

  • 用户在地址栏输入地址，发起请求（同时刷新请求）
  • 用户点击页面中的a链接，也会发起一个请求（同时刷新页面）
  • img标签，页面中如果有img标签，那么就会发起一个对此图片的请求（页面没有刷新，但是只能请求图片）类似的还有link，script标签

根据w3c标准，浏览器提供了接口[XMLHttpRequest],用js发起一个请求，并得到服务器返回的内容，页面没有刷新。

Fetch API

Fetch 是浏览器提供的原生AJAX接口，使用window.fetch函数代替以前的$.ajax,$.get,$.post,Fetch API就是浏览器提供用来代替Jquery.ajax的工具(基于Promise)

浅谈跨域

在cors技术出现以前，要实现ajax跨域通讯，利用dom中能够执行跨域请求的功能，在不依赖XHR对象的情况下也能够发送某种请求。例如图像ping和JSONP

同源策略

同源策略可以防止JavaScript发起跨域请求。源被定义为URI、主机号和端口号的组合。此策略可以防止页面的恶意脚本通过该页面的文档对象模型，访问另一个网页上的敏感数据。

1. 图像ping

图片不受“同源策略”限制。

只能发送get请求，而且无法获取响应文本。

2. JSONP

JSONP可以能够直接访问响应文本。缺点是：JSONP直接从其他域加载代码执行，如果其他域不安全，可能会在响应中夹带一些恶意代码

3. CORS

CORS跨域资源共享,CORS允许开发者发送跨域请求。CORS使用自定义的头部与服务端进行沟通，从而决定请求应该成功还是失败。

不用call和apply方法模拟实现ES5的bind方法

发表于 2018-05-01 | 分类于 js

函数调用

JS(ES5)里面有三种函数调用形式：
  • func(p1, p2)
  • obj.method(p1, p2)
  • func.call(context, p1, p2) apply 类似
func.call(context, p1, p2)

func.call(context, p1, p2)才是真正的调用方式，其他两种都是语法糖，可以等价于call形式

  • func(p1, p2)
    func.call(undefined, p1, p2)

  • obj.method(p1, p2)
    obj.method.call(obj, p1, p2)

func(p1, p2)中的this如何确定

因此我们函数的调用方式只有一种func.call(context, p1, p2)
this, 也就是call函数传递的context
func.call(),参数context是null或者undefined,那么window对象就是默认的context(++严格模式下默认context是undefined++)

obj.method(p1, p2)的this如何确定
  1. this就是call函数时，传入的第一个参数。
  2. 如果你的函数调用形式不是call形式，请按照[转换代码]将其转换为call形式。

call,apply,bind方法

call()和apply()

call和apply都是可以改变上下文中this并且立即执行这个函数
apply方法也可以改变this的指向，apply也可以有多个参数，但是第二个参数必须是一个数组
call和apply方法的第一个参数如果是null，那么this指向的是window对象

bind()

bind和call,apply方法不同的地方，实际上就是bind方法返回一个修改过后的函数
bind可以有多个参数，并且参数可以执行的时候再次添加，但是，参数是按照形参的顺序进行的
bind函数可以让对应的函数想什么时候调用就什么时候调用，并且将参数在执行的时候添加

  • W3C绑定事件，element.addEvenListener(event,listenr,useCapture).

  • IE绑定事件,element.attachEvent(event,function)

  • 事件的代理/委托

    事件代理/委托是将事件监听器添加到父元素，而不是每一个子元素单独设置事件监听器。当触发子元素时，事件会冒泡到父元素，监听器就会触发。这种技术的好处是：

    1. 内存占用减少，因为只需要一个父元素的事件处理程序，而不必为每个后代都添加事件处理程序。
    2. 无需从已删除的元素中解绑处理程序，也无需将处理程序绑定到新元素上。
